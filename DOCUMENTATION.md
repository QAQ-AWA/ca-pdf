## 按角色推荐路线

### 新用户
1. 阅读 [README](./README.md) 了解产品能力与快速开始。
2. 按照 [USER_GUIDE.md](./USER_GUIDE.md) 完成证书、签章等核心操作。
3. 如果遇到问题，参考 [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) 的排查指引。

### 开发者
1. 通过 [README](./README.md) 熟悉整体背景与 Quick Start。
2. 在 [DEVELOPMENT.md](./DEVELOPMENT.md) 完成环境准备、规范学习。
3. 深入阅读 [ARCHITECTURE.md](./ARCHITECTURE.md) 理解系统设计与技术栈。
4. 结合 [API.md](./API.md) 进行接口联调或二次开发。

### 管理员 / 运维
1. 查看 [README](./README.md) 获取组件概览和部署入口。
2. 按 [DEPLOYMENT.md](./DEPLOYMENT.md) 执行部署、配置环境变量。
3. 根据 [SECURITY.md](./SECURITY.md) 完成安全加固与合规检查。
4. 使用 [USER_GUIDE.md](./USER_GUIDE.md) 和 [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) 支持内部用户。

### 贡献者
1. 阅读 [README](./README.md) 与 [DOCS_USAGE_GUIDE.md](./DOCS_USAGE_GUIDE.md) 熟悉文档体系。
2. 按 [CONTRIBUTING.md](./CONTRIBUTING.md) 设置工作流并了解提交要求。
3. 参考 [DEVELOPMENT.md](./DEVELOPMENT.md) 获取开发细节、调试技巧。
4. 在提交前同步 [CHANGELOG.md](./CHANGELOG.md) 与相关专题文档。

## 更新与维护

- 文档统一遵循 README 中的“📘 文档维护”策略，并在变更后 7 天内完成同步。
- 每次重要功能或配置更新，需同步修改对应章节、更新 [CHANGELOG.md](./CHANGELOG.md)，并在 [docs/MAINTENANCE_LOG.md](./docs/MAINTENANCE_LOG.md) 登记。
- 使用 PR 模板中的“文档检查清单”记录状态，必要时在描述中说明未更新原因。
- 在 CI 中运行 `scripts/validate-docs.sh` 以校验链接、API 对齐与示例语法，并配合 `documentation-guardrails` 工作流自动拦截遗漏。
- 至少每季度进行一次手动巡检，重点关注链接有效性、示例可运行性与术语一致性。
