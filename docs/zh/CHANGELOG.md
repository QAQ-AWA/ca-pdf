## [1.0.0] - 2024-11-XX

### MVP 完整版本 - 自托管 PDF 电子签章平台正式发布

#### Added (新增功能)

**认证与授权系统**
- JWT Token + Refresh Token 认证机制，支持 Token 自动轮换
- 基于角色的访问控制（RBAC），支持 Admin 和 User 两级权限
- 登录速率限制（防暴力破解），Refresh Token 安全刷新机制
- 管理员首次启动自动创建默认账户功能

**核心证书管理功能**
- 自签根 CA 证书生成（支持 RSA-4096/EC-P256 算法）
- 基于根 CA 的用户证书签发功能，返回 PKCS#12 捆绑包
- 支持导入外部 PKCS#12 格式证书到系统
- 证书生命周期完整管理（查看、吊销、CRL 生成）
- 证书状态追踪（有效、过期、吊销、暂停）
- 证书吊销列表（CRL）自动生成与更新

**PDF 电子签章核心功能**
- 单个 PDF 签章 - 支持位置选择和签章样式定制
- 批量 PDF 签章 - 同时处理多个文件（默认最多 10 个）
- 可见签章 - 在 PDF 指定位置显示签名框（含证书信息和时间戳）
- 不可见签章 - 嵌入数字签名但不改变文档视观感
- 签章元数据记录 - 支持记录签章原因、地点、联系方式
- RFC3161 时间戳服务（TSA）集成
- 长期验证（LTV）- 在签名中嵌入验证材料实现长期有效

**签章验真系统**
- 多签名检验 - 一次上传可验证文件中的所有数字签名
- 有效性判断 - 清晰的状态标识（有效/无效/未签名）
- 信任链验证 - 证书链完整性和信任状态判断
- 时间戳检查 - 显示签名时间和时间戳有效性
- 修改检测 - DocMDP 级别文件完整性判断

**审计日志系统**
- 完整操作记录 - 证书生成、签章、吊销、登录、访问等全方位追踪
- 元数据捕获 - 记录执行人、IP 地址、User-Agent、操作时间
- 多维度查询功能 - 按用户、事件类型、日期范围等筛选
- 审计日志查看界面（管理员专用）
- 不可篡改记录设计

**企业印章管理**
- 企业印章上传和管理（PNG/SVG 格式）
- 加密存储机制 - 印章文件使用主密钥加密保护
- 印章版本管理 - 支持更新且保留历史版本
- 批量应用 - 在批量签章时应用同一印章到多份文档

**加密存储系统**
- 基于 Fernet 的私钥加密存储
- 敏感数据（私钥、印章等）加密保护
- 主密钥离线备份和恢复机制

**后端 API 端点（共 20 个）**
- 认证：5 个端点（登录、登出、刷新、获取用户信息、管理员 ping）
- 证书管理：8 个端点（生成 CA、查询、签发、导入、吊销、CRL 等）
- PDF 签章：3 个端点（单个签章、批量签章、验签）
- 审计日志：1 个端点（查询日志）
- 系统：1 个端点（健康检查）

**前端应用**
- 现代化 React + TypeScript 前端应用
- 登录认证页面 - JWT 令牌管理
- 证书管理工作台 - 完整的证书生命周期操作界面
- PDF 签章工作台 - 直观的单/批量签章操作流程
- 签章验真中心 - 上传验证签章有效性
- 审计日志查看器 - 多维度查询和过滤
- 系统概览与设置 - 用户信息和系统配置

**技术基础**
- FastAPI 异步 Web 框架
- SQLAlchemy 2.0 ORM 框架
- PostgreSQL（生产）和 SQLite（开发）支持
- Alembic 数据库迁移管理
- pyHanko PDF 签章库
- cryptography 密码学库
- python-jose JWT 令牌管理
- Passlib + bcrypt 密码处理

**部署与基础设施**
- Docker 容器化支持
- Docker Compose 一键部署
- Traefik 反向代理和 SSL/TLS 支持
- Let's Encrypt 证书自动续期
- Nginx 前端服务器配置

**文档与指南**
- 完整的中文 README 文档（16KB）
- 部署指南（DEPLOYMENT.md）
- 项目进度分析（PROJECT_PROGRESS_ANALYSIS.md）

## 升级指南

### 从 v0.x 升级到 v1.0.0

本版本为 MVP 完整版本，包含所有核心功能。升级步骤如下：

#### 1. 备份现有数据
```bash
# 备份 PostgreSQL 数据库
pg_dump postgresql://user:pass@host:5432/db > backup.sql

# 备份加密主密钥
cp ${ENCRYPTED_STORAGE_MASTER_KEY} master_key.bak
```

#### 2. 更新依赖
```bash
# 后端依赖
cd backend
poetry install

# 前端依赖
cd ../frontend
npm install
```

## 已知问题与限制

### 当前版本已知问题

1. **问题：Seal API 尚未实现**
   - 描述：企业印章管理 API 端点（v1.0.0 之前）
   - 影响：前端印章管理功能受限
   - 临时解决：使用默认签章样式，不应用企业印章
   - 预计修复：v1.1.0

2. **问题：用户管理功能不完整**
   - 描述：缺少用户创建、编辑、删除等管理端点
   - 影响：无法在系统中添加多用户
   - 临时解决：手动在数据库中创建用户账户
   - 预计修复：v1.1.0

3. **问题：大文件签章可能超时**
   - 描述：超过 30MB 的 PDF 签章处理可能超时
   - 影响：签章任务失败
   - 临时解决：减小文件大小或增加超时时间
   - 预计修复：v1.2.0

4. **问题：批量签章并发限制**
   - 描述：默认最多同时处理 10 个文件
   - 影响：大批量签章需要分批处理
   - 临时解决：分批提交签章任务
   - 预计修复：v1.3.0 （配置化限制）

5. **问题：移动设备支持不完整**
   - 描述：前端在小屏幕设备上体验不佳
   - 影响：移动设备用户体验下降
   - 临时解决：使用桌面设备操作
   - 预计修复：v1.4.0

## 重大功能说明

### 认证与授权
- **JWT Token 机制**：采用标准的 JWT Token + Refresh Token 方案
- **Token 轮换**：每次刷新都会颁发新的 Token，提升安全性
- **RBAC 权限控制**：支持管理员和普通用户两个权限级别
- **防暴力破解**：登录端点限流防止密码猜测

### 证书管理系统
- **根 CA 生成**：支持自签根证书，可选 RSA-4096 或 EC-P256 算法
- **证书签发**：基于根 CA 签发用户证书，返回 PKCS#12 格式
- **证书导入**：支持导入外部证书到系统
- **证书吊销**：支持即时吊销证书，自动生成 CRL
- **CRL 管理**：自动生成和维护证书吊销列表

### PDF 电子签章
- **单个签章**：支持对单个 PDF 文件进行数字签名
- **批量签章**：支持批量处理多个 PDF 文件
- **可见签章**：在指定位置显示签名框，包含时间戳和证书信息
- **不可见签章**：嵌入签名但不改变 PDF 视观感
- **时间戳支持**：集成 RFC3161 兼容的 TSA 时间戳服务
- **长期验证**：在签名中嵌入 CRL/OCSP 信息实现长期有效性

### 签章验真
- **多签名验证**：支持验证 PDF 中的所有数字签名
- **有效性判断**：清晰的状态标识（有效/无效/未签名）
- **信任链验证**：验证证书链的完整性和信任状态
- **时间戳检查**：验证时间戳的有效性和有效期
- **修改检测**：检测文件签署后是否被修改

### 审计日志
- **完整记录**：记录所有关键操作（证书生成、签章、吊销、登录等）
- **元数据记录**：包括操作人、IP 地址、User-Agent、时间戳等
- **多维度查询**：支持按用户、事件类型、日期范围等查询
- **导出功能**：支持导出为 CSV/JSON 格式
- **不可篡改**：使用数据库记录确保日志不可篡改

### 用户管理
- **多用户支持**：组织内多个用户可同时使用系统
- **权限隔离**：不同用户之间数据隔离
- **密码安全**：使用 bcrypt 加密存储，支持复杂度检查
- **状态管理**：支持启用/禁用用户账户

### 企业印章
- **格式支持**：支持 PNG（推荐透明背景）和 SVG 矢量格式
- **大小限制**：默认不超过 1 MB，可通过环境变量配置
- **加密存储**：印章文件使用主密钥加密存储
- **版本管理**：支持更新印章，旧版本保留但不使用
- **批量应用**：在批量签章时应用同一印章到多份文档

## 迁移示例

### 迁移示例：API 端点参数变更

如果在将来的版本中改变 API 参数格式，迁移步骤示例如下：

**假设场景**：PDF 签章参数格式从平面结构变更为嵌套结构

**旧格式**（v1.0.0）：
```json
POST /api/v1/pdf/sign
{
  "file": "<file>",
  "certificate_id": "uuid",
  "signature_location": "0,0,100,100",
  "visible": true,
  "reason": "Approval"
}
```

**新格式**（v1.1.0 假设）：
```json
POST /api/v1/pdf/sign
{
  "file": "<file>",
  "certificate_id": "uuid",
  "appearance": {
    "visible": true,
    "location": {
      "x": 0,
      "y": 0,
      "width": 100,
      "height": 100
    }
  },
  "metadata": {
    "reason": "Approval"
  }
}
```

**迁移步骤**：
1. 更新 API 请求代码，使用新的参数格式
2. 测试签章功能，确保结果一致
3. 如使用 API 客户端库，更新库版本
4. 更新自动化脚本中的 API 调用
