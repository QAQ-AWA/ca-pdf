## FAQ

### General Questions

**Q: What is ca-pdf?**
A: ca-pdf is a self-hosted PDF digital signature platform. See [README](./index.md) for details.

**Q: How do I get started?**
A: Start with the [README](./index.md), then follow the quick start section.

**Q: Is ca-pdf free?**
A: Yes, ca-pdf is open source under the MIT License.

**Q: Can I use it in production?**
A: Yes. See [Deployment Guide](./DEPLOYMENT.md) for production recommendations.

### Usage Questions

**Q: How do I sign a PDF?**
A: See [User Guide - Document Signing](./USER_GUIDE.md#document-signing).

**Q: How do I manage certificates?**
A: See [User Guide - Certificate Management](./USER_GUIDE.md#certificate-management).

**Q: Can I sign documents programmatically?**
A: Yes, use the REST API. See [API Documentation](./API.md).

### Deployment Questions

**Q: What are the system requirements?**
A: See [Deployment Guide - System Requirements](./DEPLOYMENT.md#system-requirements).

**Q: How do I deploy to production?**
A: See [Deployment Guide - Production Deployment](./DEPLOYMENT.md#production-deployment).

**Q: What database should I use?**
A: PostgreSQL is recommended for production. See [Deployment Guide - Database Configuration](./DEPLOYMENT.md#database-configuration).

### Security Questions

**Q: How secure is ca-pdf?**
A: Very secure. See [Security Guide](./SECURITY.md) for details.

**Q: How do I protect my keys?**
A: See [Security Guide - Key Management](./SECURITY.md#key-management).

**Q: What encryption is used?**
A: AES-256 for storage, TLS 1.2+ for transit. See [Security Guide](./SECURITY.md).

### Development Questions

**Q: How do I set up a development environment?**
A: See [Development Guide - Environment Setup](./DEVELOPMENT.md#environment-setup).

**Q: How do I run tests?**
A: See [Development Guide - Backend Testing](./DEVELOPMENT.md#backend-development).

**Q: How can I contribute?**
A: See [Contributing Guide](./CONTRIBUTING.md).
