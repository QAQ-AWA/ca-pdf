## Building Documentation

The documentation site is built using MkDocs with Material theme and i18n plugin support.

### Prerequisites

```bash
pip install mkdocs mkdocs-material mkdocs-i18n
```

### Local Development

```bash
# Navigate to project root
cd /home/engine/project

# Serve documentation locally (with live reloading)
mkdocs serve

# Open browser to http://localhost:8000
```

### Building

```bash
# Generate static site
mkdocs build

# Output is in ./site/ directory
```

## Configuration

The documentation is configured in `mkdocs.yml` at the project root with:

- **Theme**: Material Design
- **i18n Plugin**: Enabled for Chinese (zh) and English (en)
- **Default Language**: Chinese (zh)
- **Search**: Multi-language search support
- **Features**: Code copy button, tabs, collapsible sections, etc.

## Language Switching

When viewing the documentation site:

1. Default language is Chinese (中文)
2. Use the language selector in the top-right corner to switch to English
3. Each page has both language versions
4. URL structure: `/{language}/{page}/`

## Deployment

Documentation is automatically deployed to GitHub Pages using GitHub Actions workflow (`.github/workflows/docs.yml`):

- **Trigger**: Push to `main` branch with changes to `docs/`, `mkdocs.yml`, or workflow file
- **Build**: Python 3.11 environment
- **Deployment**: Published to `gh-pages` branch
- **URL**: https://qaq-awa.github.io/ca-pdf/

## Writing Documentation

### Markdown Format

All documentation files are in Markdown format with these extensions:

- Admonition (note, warning, tip, etc.)
- Code syntax highlighting with copy button
- Tables
- Tabs (for multi-version content)
- Emoji support
- Task lists

### Example Admonition

```markdown
!!! note "Title"
    This is a note.

!!! warning "Warning"
    This is a warning.

!!! tip "Helpful tip"
    This is a helpful tip.
```

### Example Code Block

```markdown
\`\`\`python
# Code block with automatic syntax highlighting
def example():
    pass
\`\`\`
```

### File Naming

- Use uppercase for document names: `API.md`, `DEPLOYMENT.md`
- Use lowercase for directory names: `docs/zh/`, `docs/en/`
- Use `index.md` for home pages

## Updating Documentation

### For Chinese Documentation

Edit files in `docs/zh/`:

```bash
# Edit a file
nano docs/zh/USER_GUIDE.md

# Preview changes
mkdocs serve
```

### For English Documentation

Edit files in `docs/en/`:

```bash
# Edit a file
nano docs/en/USER_GUIDE.md

# Preview changes
mkdocs serve
```

## File Synchronization

Chinese documentation files in `docs/zh/` are the primary source and typically contain the complete content. English documentation files in `docs/en/` may include:

- Translations of Chinese content
- Skeleton/stub files with references to Chinese versions for detailed sections
- English-specific examples or clarifications

## Navigation

Navigation is automatically generated by the mkdocs-i18n plugin based on:

1. Directory structure
2. File names
3. mkdocs.yml configuration

The plugin automatically detects language-specific directories and creates separate navigation menus for each language.

## Verification

To verify documentation builds correctly:

```bash
# Build documentation without serving
mkdocs build --strict

# Check for warnings or errors
# Output site is in ./site/
```

To contribute documentation improvements:

1. Fork the repository

3. Edit files in appropriate language directory
4. Test locally: `mkdocs serve`
5. Submit pull request
